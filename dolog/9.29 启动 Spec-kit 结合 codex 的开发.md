## Spec-kit 攻略

#### 基本逻辑
- 可以用 GPT5 做 constitution、specify、plan、tasks，然后让 Sonnet 4 来写代码（implement）并更新 tasks。
- 所有的 spec 文档都存储在 md 文件中，你可以随时让任意模型重新 run plan 和 tasks，或者写代码。
- implement 是最长的工作周期，但我们值得在 spec 阶段多轮打磨。
- spec 文档是给你**手动修改**的，不需要全用 llm 控制。

#### 2. /specify
![|450x152](https://img.evf.cc/ipic/2025-09-30-061653.png)
- 描述需求：你在 build 什么，但不进入任何技术细节。
- 它会创建一个新的 branch，【**这非常重要**】
	- 因此开启一个新的 bugfix、一个新的技术尝试，都建议用 /specify

- 建议结合 git 管理工具使用，比如 gitbutler，这样一旦 llm 不符合预期，就可以直接 discard，不要 commit，但 spec 不动。有一些改动可以 git stash。
	- 进入 implement 阶段时，事情会变得模糊易变。spec 可以 commit 进去，但 code change 需要灵活。如果出问题，修改 spec，然后 discard 代码，然后重新运行。
	- 所以要非常熟悉 branching、stage、迭代 stage 等 git 操作。
