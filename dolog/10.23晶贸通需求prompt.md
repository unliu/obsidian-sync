# 总文档
请基于给出的功能清单 excel 文件和下列描述和要求，输出完整、细致的功能需求文档。要求按 1.，1.2,1.2.1 这样的分级标题层级化梳理。

描述和要求：

「晶贸通」是一个跨境综合服务平台，需要支持跨境电商小商户在微信小程序、PC web 上使用，超级管理员在「PC 端管理后台」管理。注册方式要求用手机号 + 验证码注册（或微信小程序授权手机号）。

商户注册后，首先要求进行工商注册信息填写，如果还没有注册企业的，可以选择使用「工商注册服务」进行「工商注册申请」，也可以选择等待商户邀请。

已经填写了工商注册信息、或进行了「工商注册申请」的用户，可以通过手机号邀请其他用户加入当前企业，邀请身份可以是「商户管理员」或「普通成员（仅查看）」。被邀请的用户可以选择「接收邀请」或「拒绝」，接受邀请后，将登录到商户中。如果是「商户管理员」，将具备和创建者一样的权限；如果是「普通成员（仅查看）」，则可以查看商户的所有信息，但无法创建和编辑数据。

全局要求：
- 前端组件使用 ant design pro 框架，小程序使用 ant design mini 框架，ai 交互要简单做，使用 ant design x 框架。相关交互设计需要符合 ant design 的组件约束和设计规范。
- 本期是 MVP 产品，所有业务都只考虑主流程、正流程，不需要太多考虑逆向流程和 corner case。
- 所有的「申报/申请」都是商户管理员操作，且都会变成超级管理员的「我的待办」，超级管理员可以看到所有单据，超级管理员面对的各个模块都有「待处理」单据 tab，点击逐一处理。超级管理员可以编辑单据的数据、给每条数据切换状态，推动业务向下流转。
- 所有业务变更，需要商户知晓的，都通过「站内信」（本质上是站内通知）向商户的管理员/普通成员展示，同时通过微信小程序或服务号模版消息向商户下的所有微信已绑定用户发送通知。
- 所有需要人工上传 excel 数据的入口，都需要提供 excel 模版下载，并要求用户在模版上修改再上传。
- 需要支持超级管理员「一键切换到任意用户」功能，即：点击头像-切换到任意用户-填写用户手机号-点确定，完成切换，此时相当于登录了对应用户。唯一的区别：切换过去后还需要保持管理员的登录授权，支持点击头像-切回到管理员。
- 所有功能模块需要支持通过全局搜索框，快速搜索、点击前往。


商户首页：
- 展示工商注册、跨境物流、跨境申报、出口报关、出口退税、收结汇、培训服务、交易平台，一共八大模块的入口。如果没有完成工商注册信息填写，则「跨境物流、跨境申报、出口报关、出口退税、收结汇、交易平台」这几个入口置灰不可点击，桌面的 hover 或小程序点击提示「请在完善工商注册信息后操作」。
- 展示所有的「业务变更」动态
- 按模块展示「我的待办」，如果是出口报关、退税这样的多个单据的业务，要展示不同待办事项的数量。如：出口报关会展示形如「电子订单报送(2) - 清单总分单报送(3) - 汇总申请单报送(1)  」的入口
