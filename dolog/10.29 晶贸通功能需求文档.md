## 1. 项目概述

### 1.1 项目背景

- 「晶贸通」是一个跨境综合服务平台，旨在为跨境电商小商户提供一站式的跨境贸易服务解决方案。平台支持微信小程序和 PC web 端使用，同时提供 PC 端管理后台供超级管理员进行系统管理。
    
- v1.0 版本（https://www.jmtkj.com/）甲方反馈存在的问题：
    
    - “没有审美”
        
    - 综合平台没有首页
        
    - 没有搜索框，找模块不便
        
    - 切换账户不便
        
    - 重复信息不能自动调用

### 1.2 项目目标

  

- 为跨境电商小商户提供工商注册、跨境物流、跨境申报、出口报关、出口退税、收结汇、培训服务、交易平台等八大核心服务
    
      
    
- 实现业务流程的数字化、标准化管理
    
      
    
- 提供实时的业务状态跟踪和通知服务
    
      
    
- 支持多角色权限管理和业务协作
    
      
    

### 1.3 目标用户

  

- **商户用户**：跨境电商小商户，包括企业管理员和普通成员

## 3. 用户角色与权限

### 3.1 用户角色定义

#### 3.1.1 商户管理员

- **权限范围**：拥有商户的全部操作权限
    
- **主要职责**：工商注册信息填写、业务申请提交、团队成员管理、业务状态查看
    
      
    

#### 3.1.2 普通成员（仅查看）

- **权限范围**：只能查看商户的业务信息，无法进行创建和编辑操作
    
- **主要职责**：业务进度查看、通知接收
    
      
    

#### 3.1.3 超级管理员

- **权限范围**：拥有系统的全部管理权限
    
- **主要职责**：商户审核、业务单据处理、系统参数配置、用户管理
    
      
    

### 3.2 权限管理机制

![](https://bfvorgoabw.feishu.cn/space/api/box/stream/download/asynccode/?code=NWUyNWVmMWU4MTM1ZWRiZmQ2MzE1MTFkNzM4MmY5YzZfN0tZdjhCamRpZHFFYnA3MUhKQVFHSHRVVFA4eHpITXFfVG9rZW46UEdzQWJyN3lVb1l3N2Z4dkY0eGNxc0lxbnhmXzE3NjE3MzUwMjg6MTc2MTczODYyOF9WNA)

#### 3.2.1 基于角色的访问控制（RBAC）

- 系统采用 RBAC 权限模型
    
- 每个用户可关联多个角色
    
- 权限继承自角色
    
      
    

#### 3.2.2 数据权限控制

- 商户用户只能访问本商户的数据
    
- 超级管理员可以访问所有商户的数据
    
- 支持数据的只读/可写/超级管理权限

## 9. 非功能性需求

#### UX/UI 升级：

（1）简化核心业务操作流程（如工商注册、报关申报等）；

（2）业务界面 “向导化”（分步指引）与 “情景化”（按用户类型显示功能）；

（3）植入操作指导（如悬浮提示、视频教程链接）；

（4）验收标准：委托方 5 名操作人员（覆盖各业务模块）测试满意度≥90 分（满分 100）。

#### 性能要求：

（1）API 响应：95% 请求响应时间＜2 秒，剩余 5%≤3 秒；

（2）并发支持：≥100 用户同时在线操作，页面加载时间≤2 秒，无卡顿；

（3）数据处理：单日支持≥10 万条订单数据处理，无数据丢失。

#### 安全性要求：

（1）认证授权：采用 OAuth 2.0 或 JWT 进行 API 认证，支持角色分级（用户 / 客服 / 管理员）；

（2）数据传输：所有 API 通信使用 HTTPS（TLS 1.2 及以上）；

（3）验收标准：开发的系统经甲方指定第三方机构验收合格，系统中无被验证可利用的高危漏洞（CVSS≥7.0）。

#### 可靠性要求：

（1）系统可用性≥99.5%（年度故障累计≤4.38 小时）；

（2）灾备备份：数据库主从同步，故障自动切换（切换时间≤5 分钟）。

#### 可维护性要求：

（1）API 文档：提供 apifox 格式在线文档，含接口地址、参数、示例、错误码；

（2）小程序部署：完成微信小程序部署、认证及后台设置，需提交《晶贸通小程序后台操作说明书》；

（3）服务器搭建：按甲方要求搭建自有服务器（详见设备参数及要求），完成系统部署与环境调试，提供《服务器操作手册》。
